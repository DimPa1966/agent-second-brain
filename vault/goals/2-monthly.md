# Monthly Focus — Февраль 2026 (оставшиеся 10 дней)

## Top 3 Priorities

### Priority 1: Разобраться с OpenClaw и его применением в стартапе

**Why it matters:** Определить ключевой инструмент для создания мультиагентной системы разметки → годовая цель: MVP к концу года.

**Key Actions:**
- [ ] Изучить документацию и возможности OpenClaw
- [ ] Определить, как OpenClaw может использоваться для построения агентов разметки
- [ ] Сделать первые эксперименты — попробовать простую задачу через OpenClaw

**Definition of Done:** Чёткое понимание, подходит ли OpenClaw как основной инструмент для агентной системы, и план его использования.

---

### Priority 2: Углублённое изучение CVAT

**Why it matters:** CVAT — целевая платформа, на которой будут работать агенты. Без глубокого понимания платформы невозможно проектировать агентов → Q1 milestone.

**Key Actions:**
- [ ] Развернуть CVAT локально и провести ручную разметку разных типов
- [ ] Изучить API CVAT — возможности автоматизации, интеграции, подключения моделей
- [ ] Определить, какие типы разметки проще всего автоматизировать агентом

**Definition of Done:** Уверенное владение CVAT, понимание API и точек интеграции для агентной системы.

---

### Priority 3: Проработать бизнес-модель

**Why it matters:** Без описанной модели монетизации нельзя идти в CustDev в марте — нечего валидировать → годовая цель: финализировать бизнес-модель в Q1.

**Key Actions:**
- [ ] Описать варианты монетизации: за задачу, за объём, подписка
- [ ] Сравнить с ценами ручной разметки и конкурентов
- [ ] Выбрать базовую модель и подготовить гипотезы для проверки в CustDev

**Definition of Done:** Документ с описанной бизнес-моделью и ценовыми гипотезами, готовый к валидации с клиентами.

# Monthly Focus — Март 2026

## Top 3 Priorities

### Priority 1: Customer Development — валидация бизнес-модели

**Why it matters:** Без подтверждённого спроса и готовности платить нет смысла строить продукт → Q1 milestone: минимум 10 интервью, финализация бизнес-идеи и модели.

**Key Actions:**
- [ ] Провести 5–7 интервью с потенциальными клиентами (ML-команды, дата-компании)
- [ ] Валидировать модель монетизации, проработанную в феврале
- [ ] Выяснить, какой тип разметки — самый болезненный и за что готовы платить
- [ ] Зафиксировать выводы: кто клиент, какая задача, сколько готовы платить

**Definition of Done:** Документ с результатами CustDev: подтверждённая (или скорректированная) бизнес-модель, определённый ICP (ideal customer profile).

---

### Priority 2: Первый proof-of-concept — агент разметки в CVAT

**Why it matters:** Проверка технической гипотезы — может ли AI-агент выполнять разметку в CVAT → основа для MVP в Q2.

**Key Actions:**
- [ ] Создать простейшего агента на OpenClaw / Claude Code для одного типа разметки
- [ ] Подключить агента к CVAT через API
- [ ] Провести тест: агент размечает набор данных, замерить скорость и качество
- [ ] Сравнить результат с ручной разметкой (скорость, стоимость, точность)

**Definition of Done:** Работающий proof-of-concept — агент выполняет один тип разметки в CVAT с измеримыми метриками.

---

### Priority 3: Определить первый тип разметки для MVP

**Why it matters:** Фокус на одной нише — ключ к быстрому запуску. Распыление на всё сразу убьёт скорость → Q1 milestone.

**Key Actions:**
- [ ] На основе CustDev определить, где самый большой pain и готовность платить
- [ ] На основе proof-of-concept определить, где агент показывает лучшие результаты
- [ ] Свести: пересечение "клиенту больно" + "агент справляется" = первая ниша
- [ ] Зафиксировать решение и план MVP на Q2

**Definition of Done:** Выбран конкретный тип разметки для MVP, обоснованный данными CustDev и технических тестов. Готов план на Q2.

---

## NOT Doing This Month

- Не строить полноценный MVP — только proof-of-concept одного агента
- Не пытаться автоматизировать несколько типов разметки одновременно
- Не искать инвесторов и не тратить время на питч-деки
- Не нанимать людей — всё делаю сам
- Не создавать сайт или маркетинговые материалы
